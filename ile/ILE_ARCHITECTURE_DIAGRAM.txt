╔══════════════════════════════════════════════════════════════════════════════╗
║                         DOMAIN 1: TASK-BASED LEARNING                        ║
╚══════════════════════════════════════════════════════════════════════════════╝

┌─────────────────────────────────────────────────────────────────────────────┐
│  1. TASK OUTCOME TRACKER (task_outcome_tracker.py)                          │
│  ┌────────────────────────────────────────────────────────────────────────┐ │
│  │  record_prediction(event)                                              │ │
│  │  • Store prediction with unique ID                                     │ │
│  │  • PostgreSQL: task_outcomes table                                     │ │
│  │  • Track: inputs, predicted values, timestamp                          │ │
│  └────────────────────────────────────────────────────────────────────────┘ │
│                                                                              │
│  ┌────────────────────────────────────────────────────────────────────────┐ │
│  │  record_outcome(prediction_id, actual)                                 │ │
│  │  • Match outcome to prediction (handles delayed feedback)             │ │
│  │  • Calculate learning_signal: -1.0 (very wrong) to +1.0 (very right) │ │
│  │  • Return LearningEvent for RL processing                             │ │
│  └────────────────────────────────────────────────────────────────────────┘ │
└─────────────────────────┬───────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  2. REINFORCEMENT LEARNER (reinforcement_learner.py)                        │
│  ┌────────────────────────────────────────────────────────────────────────┐ │
│  │  compute_metrics(events)                                               │ │
│  │  • Accuracy, MAE, Brier score (via metrics.py)                        │ │
│  │  • Confusion matrix, precision, recall, F1                            │ │
│  │  • Calibration analysis                                               │ │
│  │  Returns: LearningMetrics object                                      │ │
│  └────────────────────────────────────────────────────────────────────────┘ │
│                                                                              │
│  ┌────────────────────────────────────────────────────────────────────────┐ │
│  │  update_policy(events)                                                 │ │
│  │  • Contextual bandit RL (via rl_engine.py)                            │ │
│  │  • Context: industry, deal size, growth rate, etc.                    │ │
│  │  • Action: risk threshold, model parameters                           │ │
│  │  • Reward: learning_signal from outcomes                              │ │
│  │  • Updates policy: best action per context                            │ │
│  └────────────────────────────────────────────────────────────────────────┘ │
└─────────────────────────┬───────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  3. MODEL UPDATE COORDINATOR (model_update_coordinator.py)                  │
│  ┌────────────────────────────────────────────────────────────────────────┐ │
│  │  propose_updates(metrics, policy)                                      │ │
│  │  • Generate update proposals from RL policy                            │ │
│  │  • Example: "Increase risk threshold for high-growth SaaS by 0.05"   │ │
│  │  • Include: expected impact, confidence, affected segments            │ │
│  └────────────────────────────────────────────────────────────────────────┘ │
│                          │                                                   │
│                          ▼                                                   │
│  ┌────────────────────────────────────────────────────────────────────────┐ │
│  │  CONSTITUTIONAL VALIDATOR (constitutional_validator.py)                │ │
│  │  • Benefit score: How much does this help humans?                     │ │
│  │  • Harm score: Could this cause harm?                                 │ │
│  │  • Human primacy check: Maintains human control?                      │ │
│  │  • Decision: APPROVED / REVIEW_REQUIRED / REJECTED                    │ │
│  └────────────────────────────────────────────────────────────────────────┘ │
│                          │                                                   │
│                          ▼                                                   │
│  ┌────────────────────────────────────────────────────────────────────────┐ │
│  │  apply_approved_updates(proposals)                                     │ │
│  │  • Update model_registry with new parameters                           │ │
│  │  • BUE reads updated configs on next prediction                        │ │
│  │  • A/B test: 10% → 50% → 100% rollout                                │ │
│  │  • Automatic rollback if performance degrades                          │ │
│  └────────────────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────────────┘

                                 OUTCOME

│  └────────────────────────────────────────────────────────────────────────┘ │

└─────────────────────────┬───────────────────────────────────────────────────┘

│  │  • Group by task type: Analysis, Planning, Monitoring, etc.           │ │

╔══════════════════════════════════════════════════════════════════════════════╗
║                     DOMAIN 6: MULTI-LLM COORDINATION                         ║
╚══════════════════════════════════════════════════════════════════════════════╝

┌─────────────────────────────────────────────────────────────────────────────┐
│  ENSEMBLE COORDINATOR (ensemble_coordinator.py)                             │
│                                                                              │
│  ┌────────────────────────────────────────────────────────────────────────┐ │
│  │  Available LLMs:                                                       │ │
│  │  • Claude Sonnet 4.5 - Best for: reasoning, analysis, technical       │ │
│  │  • GPT-4 - Best for: creative, conversational, general                │ │
│  │  • Gemini Pro - Best for: multimodal, research, long context          │ │
│  └────────────────────────────────────────────────────────────────────────┘ │
│                                                                              │
│  ┌────────────────────────────────────────────────────────────────────────┐ │
│  │  track_performance(query_event)                                        │ │
│  │  • For each query:                                                     │ │
│  │    - Query type: financial, creative, research, coding, etc.          │ │
│  │    - Model selected: which LLM was chosen                              │ │
│  │    - Quality: user rating, task success, accuracy                      │ │
│  │    - Cost: $ per query                                                 │ │
│  │    - Latency: response time (ms)                                       │ │
│  │                                                                         │ │
│  │  • Store performance by (query_type, model) pairs                     │ │
│  │  • Calculate multi-objective reward:                                   │ │
│  │    reward = 0.5*quality + 0.3*(1-cost) + 0.2*(1-latency)             │ │
│  └────────────────────────────────────────────────────────────────────────┘ │
│                          │                                                   │
│                          ▼                                                   │
│  ┌────────────────────────────────────────────────────────────────────────┐ │
│  │  update_routing_weights()                                              │ │
│  │  • Contextual bandit RL (via rl_engine.py)                            │ │
│  │  • Context: query_type, complexity, user_segment                       │ │
│  │  • Actions: route to Claude / GPT-4 / Gemini                          │ │
│  │  • Reward: multi-objective score                                       │ │
│  │  • Policy: probability distribution over models                        │ │
│  │                                                                         │ │
│  │  Example learned weights:                                              │ │
│  │  Financial Reasoning → Claude: 85%, GPT-4: 10%, Gemini: 5%           │ │
│  │  Creative Writing    → GPT-4: 80%, Claude: 15%, Gemini: 5%           │ │
│  │  Multimodal Research → Gemini: 75%, Claude: 20%, GPT-4: 5%           │ │
│  └────────────────────────────────────────────────────────────────────────┘ │
│                          │                                                   │
│                          ▼                                                   │
│  ┌────────────────────────────────────────────────────────────────────────┐ │
│  │  CONSTITUTIONAL VALIDATOR                                              │ │
│  │  • Benefit: Does routing improve quality and reduce cost?              │ │
│  │  • Harm: Could poor routing harm user experience?                      │ │
│  │  • Fairness: All models get fair evaluation (exploration)              │ │
│  └────────────────────────────────────────────────────────────────────────┘ │
│                          │                                                   │
│                          ▼                                                   │
│  ┌────────────────────────────────────────────────────────────────────────┐ │
│  │  apply_routing_updates()                                               │ │
│  │  • Update model_registry with routing weights                          │ │
│  │  • UIE reads weights on each request                                   │ │
│  │  • Selects model: 90% exploit best, 10% explore alternatives          │ │
│  │  • Results: Quality +14%, Cost -18%, Accuracy 94%                     │ │
│  └────────────────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────────────┘
